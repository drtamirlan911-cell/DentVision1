# Подключение Kaspi клиники (касса CRM)

Деньги с **кассы / расписания / карточки пациента** идут на **счёт клиники**, не на DentVision.

Платформенный Kaspi DentVision используется только для:
- Academy OS (вебинары, курсы)
- Магазин (Marketplace)
- Тариф SaaS клиники («Тариф и оплата»)

---

## Кто настраивает

Только **Руководитель** или **Администратор**:  
`CRM → Настройки клиники → Оплата на кассе (Kaspi клиники)`.

---

## Вариант A — быстро (телефон / ссылка Kaspi)

Подходит, если у клиники уже есть Kaspi для бизнеса, а API ещё нет.

1. Откройте **Настройки клиники**.
2. В блоке **Оплата на кассе** выберите режим **Телефон / ссылка Kaspi**.
3. Укажите:
   - **Название мерчанта** — как видно пациенту (например, «Стоматология Астана»).
   - **Телефон Kaspi** клиники (с которого принимаете переводы), **или**
   - **Ссылку / QR-пейлоад** из кабинета Kaspi (если есть готовая ссылка оплаты).
4. Нажмите **Сохранить**.
5. Проверка: **Касса** или **Расписание → Принять оплату → QR-оплата → Создать QR**.  
   Пациент сканирует QR → перевод на счёт клиники → администратор нажимает **Проверить оплату**.

> В этом режиме статус «оплачено» подтверждает сотрудник кассы после того, как увидел оплату в приложении Kaspi клиники.

---

## Вариант B — API-шлюз (автоматический QR)

Когда клиника подключит шлюз Kaspi Pay (ApiPay, PayBot, qrapi, прямой договор и т.п.):

1. В том же блоке выберите режим **API-шлюз**.
2. Заполните:
   - **API Base URL** (например `https://api.apipay.kz/api/v1`)
   - **API Key** (из кабинета шлюза)
   - **Webhook secret** (секрет подписи webhook)
3. Скопируйте **Webhook URL** из настроек DentVision и вставьте его в кабинет шлюза.
4. Сохраните настройки.
5. Создайте тестовый QR в кассе на 100 ₸ и оплатите с телефона пациента.

Формат webhook (клиника):

```http
POST /api/payments/callbacks/kaspi/clinic/{clinicId}
Headers:
  X-Kaspi-Signature: <shared-secret>
  # или HMAC-SHA256 hex от "externalId:status"
Body:
  { "externalId": "...", "status": "paid" }
```

---

## Что видит касса

| Способ оплаты | Куда идут деньги | QR |
|---|---|---|
| QR-оплата | Счёт клиники (ваш Kaspi/банк) | Да, после «Создать QR» |
| Наличные / карта / перевод | Учёт в CRM | Нет |
| Academy / Магазин / Тариф SaaS | Платформа DentVision | Отдельный Kaspi платформы |

Если QR на кассе не создаётся:

> «Касса клиники не подключена к своему Kaspi/банку…»

→ вернитесь в настройки и заполните Вариант A или B.

---

## Безопасность

- API Key и Webhook secret **не показываются** повторно в интерфейсе (только «задан / не задан»).
- Пустое поле при сохранении = «оставить прежний секрет».
- Не передавайте ключи клиники в поддержку DentVision без необходимости — касса клиники изолирована от платформенного мерчанта.

---

## Чеклист перед продом

- [ ] Режим A или B сохранён, статус «Подключено»
- [ ] Тестовый QR из Кассы оплачен на телефон клиники
- [ ] «Проверить оплату» пишет чек в кассе
- [ ] Для API: webhook доходит, статус payment → `paid`
- [ ] Убедились, что Academy/Shop по-прежнему на платформенном Kaspi (не трогает кассу)
